## Введение

Первые две главы были геометрическими: точки, стрелки, углы. Эта глава — алгебраическая сердцевина курса. Системы линейных уравнений (СЛАУ) — исторический исток всей линейной алгебры: матрицы (глава 4), ранг, базисы (глава 6) и обратимость выросли из одного вопроса — *как решать много линейных уравнений сразу и что вообще значит «решить»*.

Для 3D-разработчика СЛАУ — это, среди прочего: «в какой точке луч пересекает плоскость?», «как разложить вектор по осям повёрнутой системы?», «лежат ли четыре вершины в одной плоскости?». Даже задача Г2.Ч1.З6 («вырази вектор через a и b») была маленькой СЛАУ.

После главы ты будешь уметь:

- решать системы методом Гаусса — единственным по-настоящему универсальным алгоритмом этого курса;
- классифицировать системы: одно решение, ни одного, бесконечно много — и понимать геометрию каждого случая;
- проверять наборы векторов на **линейную зависимость** и вычислять **ранг** — без этих понятий не определить базис (глава 6) и обратимость матрицы (глава 4).

Требуется из пройденного: векторы и линейные комбинации (гл. 2, ч. 1), геометрия прямых и плоскостей на уровне интуиции (гл. 1).

## Часть 1. СЛАУ и метод Гаусса

### § 3.1.1. Линейное уравнение

**Определение.** *Линейное уравнение* с неизвестными x₁, …, x_n — уравнение вида

```
a₁x₁ + a₂x₂ + … + a_nx_n = b
```

где коэффициенты a₁, …, a_n и правая часть b — числа. Ключевое слово — «линейное»: неизвестные входят только в первой степени и не перемножаются между собой [S3; S1, ch. 2].

Линейны: 2x + 3y = 7; x − y + z = 0. НЕ линейны: x² + y = 1 (квадрат), xy = 4 (произведение неизвестных), sin(x) = 0.5.

**Геометрия.** Множество решений одного линейного уравнения: на плоскости (2 неизвестные) — прямая; в пространстве (3 неизвестные) — плоскость. Отсюда вся геометрическая интуиция этой главы.

### § 3.1.2. Система уравнений: две картинки

**Определение.** *Система линейных уравнений* — несколько линейных уравнений с общим набором неизвестных. *Решение системы* — набор значений неизвестных, обращающий **каждое** уравнение в верное равенство [S3; S1, ch. 2].

На систему полезно смотреть двумя способами — Стрэнг называет их *row picture* и *column picture* [S1, ch. 2; MIT 18.06, лекция 1]:

**Строчная картинка (row picture):** каждое уравнение — прямая (плоскость); решение — точка их пересечения.

**Столбцовая картинка (column picture):** система — это вопрос о линейной комбинации (гл. 2, § 2.1.4). Система

```
 x + y = 5
2x − y = 1
```

спрашивает: с какими коэффициентами x, y комбинация столбцов даёт правую часть?

```
x·(1, 2) + y·(1, −1) = (5, 1)
```

**Пример 1.** Решим эту систему сложением уравнений: (x + y) + (2x − y) = 5 + 1 ⟹ 3x = 6 ⟹ x = 2, тогда y = 3.
Проверка строчной картинки: точка (2, 3) лежит на обеих прямых. Столбцовой: 2·(1, 2) + 3·(1, −1) = (2 + 3, 4 − 3) = (5, 1). ✓

**Интуиция.** Row picture отвечает «где пересекаются условия», column picture — «как собрать цель из имеющихся векторов». Вторая картинка окажется важнее: на ней построены главы 4–6.

### § 3.1.3. Элементарные преобразования

Решать «сложением уравнений» можно систематически. **Элементарные преобразования системы**:

1. поменять два уравнения местами;
2. умножить уравнение на ненулевое число;
3. прибавить к уравнению другое, умноженное на любое число.

**Теорема.** Элементарные преобразования не меняют множества решений системы [S3].

Почему: каждое преобразование обратимо преобразованием того же типа, поэтому старая и новая системы выводимы друг из друга — любое решение одной удовлетворяет другой.

### § 3.1.4. Метод Гаусса

**Алгоритм** (прямой ход + обратный ход) [S3; S1, ch. 2]:

1. **Прямой ход**: элементарными преобразованиями обнуляем всё под «ведущими» (первыми ненулевыми) коэффициентами, двигаясь сверху вниз и слева направо, — приводим систему к *ступенчатому виду* (echelon form): каждая следующая строка начинается правее предыдущей.
2. **Обратный ход**: из последнего уравнения находим последнюю неизвестную и подставляем вверх (back substitution).

**Пример 2 (полный разбор 3×3).**

```
x + 2y +  z = 8        (1)
2x + 3y −  z = 5        (2)
x −  y + 2z = 5        (3)
```

*Прямой ход.* Убираем x из (2) и (3): (2) − 2·(1) и (3) − (1):

```
x + 2y +  z =  8       (1)
   − y − 3z = −11      (2′)
  − 3y +  z = −3       (3′)
```

Убираем y из (3′): (3′) − 3·(2′):

```
x + 2y +  z =  8       (1)
   − y − 3z = −11      (2′)
        10z =  30      (3″)
```

Ступенчатый вид получен. *Обратный ход:* z = 3; из (2′): −y − 9 = −11 ⟹ y = 2; из (1): x + 4 + 3 = 8 ⟹ x = 1.

**Ответ:** (x, y, z) = (1, 2, 3). Проверка подстановкой во все три исходных уравнения — обязательная привычка: 1+4+3=8 ✓, 2+6−3=5 ✓, 1−2+6=5 ✓.

**Пример 3 (зачем менять строки).** В системе с первым уравнением 0·x + y = 1 ведущего x нет — начинаем с перестановки уравнений. В компьютерных реализациях строки переставляют даже при ненулевом, но *маленьком* ведущем элементе (частичный выбор главного элемента, partial pivoting) — ради численной устойчивости; помни об этом, когда встретишь готовые решатели.

**Связь с практикой.** Метод Гаусса — рабочая лошадка под капотом: так вычисляют обратные матрицы (глава 4) и решают системы в физических движках и решателях ограничений. А маленькие системы 2×2–3×3 3D-программист решает постоянно: пересечение двух прямых на плоскости (два уравнения прямых), разложение вектора по паре направлений (Г2.Ч1.З6), поиск барицентрических координат точки в треугольнике (система «собери точку из вершин» — встретится при интерполяции в главе 12).

**Типичные ошибки.**

1. Арифметика знаков при вычитании строк — главный источник ошибок «на бумаге». Лекарство: проверка подстановкой (секунды) после каждого решения.
2. Умножение строки на ноль — это не элементарное преобразование: оно уничтожает уравнение и расширяет множество решений.
3. Останавливаться после прямого хода, забыв обратный: ступенчатый вид — ещё не ответ.

**Источники части**: [S1] Strang, ch. 2; MIT 18.06, лекции 1–2 (https://ocw.mit.edu/courses/18-06sc-linear-algebra-fall-2011/); [S3] Кострикин, ч. 2.

## Часть 2. Множества решений

### § 3.2.1. Три случая

**Теорема (альтернатива для СЛАУ).** Система линейных уравнений имеет либо ровно одно решение, либо ни одного, либо бесконечно много [S3; S1, ch. 2].

«Ровно два решения» у СЛАУ не бывает — и геометрия объясняет почему. На плоскости два уравнения — две прямые:

```
  единственное          нет решений           бесконечно много
  (пересекаются)        (параллельны)         (совпадают)

      \  |                 |   |                 ||
       \ |                 |   |                 ||
        \|                 |   |                 ||
         *                 |   |                 ||
         |\                |   |                 ||
```

Система, у которой решений нет, называется *несовместной* (inconsistent); у которой есть — *совместной*.

**Пример 4 (несовместность).** {x + y = 2; 2x + 2y = 5}. Вычтем из второго удвоенное первое: 0 = 1 — противоречие, решений нет. Геометрия: прямые параллельны (одинаковое направление, разные «высоты»). Сигнатура несовместности в методе Гаусса: строка вида **0 = c** с ненулевым c.

**Пример 5 (бесконечно много).** {x + y + z = 3; x − y + z = 1}. Вычитаем: 2y = 2, y = 1; остаётся одно уравнение x + z = 2 на две неизвестные. Два уравнения — две плоскости, пересекающиеся по прямой: точек-решений бесконечно много.

### § 3.2.2. Свободные переменные и общее решение

После прямого хода Гаусса переменные делятся на два сорта [S1, ch. 3; S3]:

- **ведущие** (связанные, pivot variables) — те, на которых стоит «ступенька»;
- **свободные** (free variables) — все остальные; им можно придать *любые* значения, и ведущие однозначно восстановятся.

*Общее решение* записывают, обозначив свободные переменные параметрами.

**Пример 5 (продолжение).** Свободная переменная — z (обозначим z = t). Тогда y = 1, x = 2 − t:

```
(x, y, z) = (2 − t, 1, t) = (2, 1, 0) + t·(−1, 0, 1),   t ∈ ℝ
```

Читается геометрически: **точка + t·вектор** — параметрическое уравнение прямой (язык главы 1, § 1.3.2!). Решения образуют прямую, проходящую через (2, 1, 0) вдоль направления (−1, 0, 1). Проверка при t = 0 и t = 1 подстановкой — обе точки удовлетворяют системе. ✓

**Правило подсчёта.** Число свободных переменных = (число неизвестных) − (число ступенек). Число ступенек в части 3 получит имя — *ранг*.

### § 3.2.3. Однородные системы

**Определение.** Система *однородная* (homogeneous), если все правые части — нули [S3].

Два факта:

1. Однородная система **всегда совместна**: нулевой набор x₁ = … = x_n = 0 (*тривиальное решение*) подходит всегда.
2. Вопрос только в том, есть ли **нетривиальные** решения — а это в точности вопрос «есть ли свободные переменные». В частности, если уравнений меньше, чем неизвестных, нетривиальные решения есть обязательно (ступенек не хватит на все переменные).

Однородные системы — технический фундамент следующей части: проверка линейной зависимости — это ровно вопрос о нетривиальном решении однородной системы.

**Связь с практикой.** Три случая решений — это три геометрических сценария, которые 3D-код обязан различать: луч *пересекает* плоскость в точке (одно решение), *параллелен* ей (нет решений), *лежит в ней* (бесконечно много). Робастный код пересечений всегда содержит ветку «знаменатель ≈ 0» — это ровно распознавание вырожденных случаев из этой части. Ещё пример: система «точка лежит на обеих прямых» для почти параллельных прямых численно неустойчива — маленькие ошибки входных данных сильно двигают точку пересечения.

**Типичные ошибки.**

1. Увидев строку 0 = 0, решить, что система несовместна. Наоборот: 0 = 0 — просто «пустое» уравнение, его выбрасывают; несовместность — это 0 = c ≠ 0.
2. Записать частное решение (при конкретном t) вместо общего.
3. Параметризовать ведущую переменную вместо свободной — параметризация получается неполной.

**Источники части**: [S1] Strang, ch. 2–3; [S3] Кострикин, ч. 2.

## Часть 3. Линейная зависимость и ранг

### § 3.3.1. Линейная зависимость

**Определение.** Векторы v₁, …, v_k *линейно зависимы* (linearly dependent), если существует их линейная комбинация с **не всеми нулевыми** коэффициентами, равная нулевому вектору:

```
c₁v₁ + … + c_kv_k = 0,   хотя бы один cᵢ ≠ 0
```

Иначе векторы *линейно независимы* [S3; S1, ch. 3; S2, ch. 2].

Эквивалентная формулировка (часто удобнее): векторы зависимы ⟺ один из них выражается линейной комбинацией остальных.

**Интуиция.** Зависимый набор содержит «лишний» вектор, не добавляющий нового направления: он лежит в линейной оболочке (span, гл. 2, § 2.1.4) остальных [S5, эп. 2]. Малые случаи:

- 2 вектора зависимы ⟺ коллинеарны (лежат на одной прямой);
- 3 вектора в 3D зависимы ⟺ компланарны (лежат в одной плоскости);
- любой набор, содержащий нулевой вектор, зависим (возьми c = 1 при нём и нули у прочих).

**Пример 6.** u = (1, 2, 3), v = (2, 4, 6): зависимы, ведь 2u − v = 0 (v = 2u — коллинеарны, что мы умеем видеть и через cross: u × v = 0, гл. 2, § 2.4.2).

**Пример 7.** u = (1, 0, 0), v = (0, 1, 0), w = (1, 1, 0): зависимы — w = u + v (все три лежат в плоскости z = 0). А набор u, v, (0, 0, 1) независим: комбинация c₁u + c₂v + c₃(0,0,1) = (c₁, c₂, c₃) равна нулю только при c₁ = c₂ = c₃ = 0.

### § 3.3.2. Проверка на зависимость — это однородная СЛАУ

Как проверять в общем случае? Записать определение покоординатно. Условие c₁v₁ + … + c_kv_k = 0 — это **однородная система** на неизвестные c₁, …, c_k (по уравнению на каждую координату). Дальше — § 3.2.3:

- только тривиальное решение ⟹ независимы;
- есть нетривиальное ⟹ зависимы (само решение — «рецепт» зависимости).

**Пример 8.** Зависимы ли a = (1, 1, 0), b = (0, 1, 1), c = (1, 0, −1)?
Система c₁·a + c₂·b + c₃·c = 0 покоординатно:

```
c₁      + c₃ = 0
c₁ + c₂      = 0
     c₂ − c₃ = 0
```

Гаусс: из первого c₁ = −c₃; подставим во второе: c₂ = c₃; третье: c₃ − c₃ = 0 — верно всегда. Свободная переменная c₃ = t: (c₁, c₂, c₃) = (−t, t, t). При t = 1: −a + b + c = 0, то есть **c = a − b** — зависимы. Проверка: (1,1,0) − (0,1,1) = (1, 0, −1). ✓

Полезное следствие (из § 3.2.3): **в n-мерном пространстве любые n+1 векторов линейно зависимы** — переменных в однородной системе больше, чем уравнений. Четыре вектора в 3D всегда зависимы.

### § 3.3.3. Ранг

**Определение.** *Ранг* (rank) системы векторов — максимальное число линейно независимых векторов в ней. Практически ранг вычисляется методом Гаусса: запиши векторы строками таблицы, приведи к ступенчатому виду — **ранг равен числу ненулевых строк** (ступенек) [S3; S1, ch. 3].

(В главе 4 таблица чисел получит имя «матрица», и ранг станет её характеристикой. Забегая на шаг: ранг по строкам и ранг по столбцам всегда совпадают — этот нетривиальный факт доказывается в курсах по S1/S3; мы примем его с ссылкой.)

**Интуиция.** Ранг — «сколько по-настоящему разных направлений» в наборе. Три компланарных вектора: ранг 2 — направлений на самом деле два.

**Пример 9.** Векторы (1, 2, 1), (2, 4, 2), (1, 1, 1) — строки таблицы:

```
1 2 1        1 2 1        1  2  1
2 4 2   →    0 0 0   →    0 −1  0     (переставили строки: ступеньки по порядку)
1 1 1        0 −1 0       0  0  0
```

Две ненулевые строки ⟹ **ранг 2**: второй вектор — удвоенный первый, «нового направления» не даёт.

### § 3.3.4. Критерий совместности (теорема Кронекера–Капелли)

**Теорема (Кронекера–Капелли).** Система линейных уравнений совместна тогда и только тогда, когда ранг таблицы коэффициентов равен рангу расширенной таблицы (коэффициенты + столбец правых частей) [S3].

**Интуиция на столбцовой картинке.** Система разрешима ⟺ вектор правой части лежит в линейной оболочке столбцов ⟺ добавление его к столбцам не увеличивает число независимых направлений.

**Пример 10.** Для несовместной системы из примера 4 ({x + y = 2; 2x + 2y = 5}): коэффициенты — строки (1, 1), (2, 2), ранг 1; расширенные строки (1, 1, 2), (2, 2, 5) независимы (вторая не кратна первой), ранг 2. Ранги различны ⟹ решений нет — согласуется с «0 = 1» из примера 4.

**Связь с практикой.** Линейная зависимость в 3D-коде — синоним **вырожденности**, и её надо ловить:

- треугольник с коллинеарными вершинами (B−A и C−A зависимы) имеет нулевую площадь и нулевую нормаль — cross из гл. 2 даёт 0, освещение ломается;
- тройка зависимых векторов не может служить осями системы координат/базисом (глава 6) — матрица из них необратима (глава 4);
- `cross(forward, up)` при коллинеарных аргументах (камера смотрит вертикально — Г2.Ч4.З10) — тот же сюжет: зависимость = потеря направления.

**Типичные ошибки.**

1. «Все векторы попарно неколлинеарны ⟹ независимы» — неверно с трёх векторов: u, v, u+v попарно неколлинеарны, но зависимы (пример 7).
2. Путать тривиальное решение с независимостью наоборот: нулевая комбинация с нулевыми коэффициентами есть *всегда* — независимость означает, что она **единственная**.
3. Считать ранг по исходной таблице «на глаз», без приведения к ступенчатому виду.

**Источники части**: [S1] Strang, ch. 3; [S2] Axler, ch. 2; [S3] Кострикин, ч. 2; [S5] 3Blue1Brown, эп. 2.

## Резюме главы

- СЛАУ можно читать двумя способами: пересечение прямых/плоскостей (row picture) и подбор линейной комбинации столбцов (column picture).
- Метод Гаусса: элементарными преобразованиями (не меняющими решений) — к ступенчатому виду, затем обратный ход. Всегда делай проверку подстановкой.
- Решений бывает 0, 1 или ∞. Несовместность — строка 0 = c ≠ 0; бесконечность — свободные переменные; общее решение — «точка + параметры·векторы».
- Однородная система всегда имеет тривиальное решение; нетривиальные есть ⟺ есть свободные переменные (в частности, всегда, когда неизвестных больше, чем уравнений).
- Линейная зависимость = нетривиальная комбинация, дающая 0 = «лишний вектор без нового направления». Проверка — однородная СЛАУ. В n-мерии любые n+1 векторов зависимы.
- Ранг — число независимых направлений = число ступенек после Гаусса. Совместность ⟺ ранги таблицы и расширенной таблицы равны (Кронекер–Капелли).
- В 3D зависимость = вырожденность: нулевые нормали, негодные «оси», необратимые матрицы (глава 4).

**Дальше — глава 4 «Матрицы»**: таблицы коэффициентов, которыми мы уже фактически пользовались, станут полноправными объектами со своей арифметикой — сложением, умножением, детерминантом и обращением. Именно матрицы повезут на себе все трансформации 3D-мира.
