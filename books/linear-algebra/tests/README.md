# Верификация сборника решений

Три скрипта перепроверяют числовые ответы решебника (`book/solutions/`) реально выполняемым кодом — требование правила 4.4 плана (`self-education-book-create-plan.md`): никаких догадок, расчёты перепроверены.

## Запуск

```bash
node verify_ch1_5.js     # главы 1–5: векторы, СЛАУ, матрицы, преобразования (~90 ассертов)
node verify_ch6_9.js     # главы 6–9: базисы, TRS, look-at, проекции, сквозной пайплайн (~50)
node verify_ch10_12.js   # главы 10–12: Родриг, кватернионы, собственные значения, освещение
```

Каждый скрипт печатает `ALL CHECKS PASSED` или список расхождений (`FAIL <ID задания>: got … want …`) и возвращает ненулевой код выхода при провале.

## Что внутри

- Ассерты привязаны к идентификаторам заданий (`Г{глава}.Ч{часть}.З{номер}`) — по имени провалившегося теста сразу видно, какое решение проверять.
- `verify_ch10_12.js` дополнительно содержит **эталонную реализацию капстоун-библиотеки** главы 12 (Г12.Ч4.З1–З6): vec3, mat4 (column-major, как GLSL), quat, lookAt, perspective — с ней сверяется твоя собственная библиотека из итогового проекта. Там же — сквозной тест пайплайна «вершина (1,1,0) → пиксель (457, 343)».

## Когда запускать

- После любой правки решений или условий заданий (регрессия чисел).
- При выполнении капстоуна (гл. 12, ч. 4) — как источник эталонных значений.
