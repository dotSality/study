## Часть 1. Координаты на плоскости и в пространстве

### Г1.Ч1.З1

**Условие.** На числовой оси отмечены точки с координатами a = −5, b = 3, c = 7. Найди расстояния d(a,b), d(b,c), d(a,c). Проверь: выполняется ли d(a,c) = d(a,b) + d(b,c), и почему?

**Решение.** Расстояние на оси — модуль разности координат (§ 1.1.1):
d(a,b) = |3 − (−5)| = 8; d(b,c) = |7 − 3| = 4; d(a,c) = |7 − (−5)| = 12.
Проверка: 8 + 4 = 12 — выполняется. Причина: точка b лежит **между** a и c (−5 < 3 < 7), поэтому путь из a в c проходит через b без «возвратов». Если бы b лежала вне отрезка [a, c], сумма была бы больше.

**Ответ:** 8, 4, 12; равенство выполняется, потому что b между a и c.

### Г1.Ч1.З2

**Условие.** Определи, в каком квадранте (или на какой оси) лежит каждая точка: (3, −2), (−1, −1), (0, 4), (−5, 0), (2, 2).

**Решение.** Смотрим на знаки координат: I квадрант (+,+), II (−,+), III (−,−), IV (+,−); ноль в координате — точка на оси.

**Ответ:** (3, −2) — IV квадрант; (−1, −1) — III; (0, 4) — на оси Oy; (−5, 0) — на оси Ox; (2, 2) — I.

### Г1.Ч1.З3

**Условие.** Вычисли расстояния между парами точек на плоскости: A(0,0) и B(3,4); C(−1,2) и D(2,−2); E(5,5) и F(5,−5).

**Решение.** По формуле d = √(Δx² + Δy²):
- A, B: √(3² + 4²) = √25 = **5**;
- C, D: Δx = 2−(−1) = 3, Δy = −2−2 = −4; √(9+16) = **5**;
- E, F: Δx = 0, Δy = −10; √(0+100) = **10** (точки на одной вертикали — расстояние равно |Δy|).

**Ответ:** 5; 5; 10.

### Г1.Ч1.З4

**Условие.** Вычисли расстояния в пространстве: A(1,0,0) и B(0,1,0); C(2,3,6) и O(0,0,0); D(1,1,1) и E(4,5,1).

**Решение.** d = √(Δx² + Δy² + Δz²):
- A, B: √(1+1+0) = **√2 ≈ 1.414**;
- C, O: √(4+9+36) = √49 = **7**;
- D, E: Δ = (3, 4, 0); √(9+16+0) = **5** (z совпали — задача «сложилась» в плоскую, как в примере 2 главы).

**Ответ:** √2; 7; 5.

### Г1.Ч1.З5

**Условие.** Какая из точек P(6, 2, 3) и Q(4, 4, 4) ближе к началу координат? Реши, не извлекая квадратных корней, и объясни, почему так можно.

**Решение.** Сравниваем **квадраты** расстояний: d²(O,P) = 36+4+9 = 49; d²(O,Q) = 16+16+16 = 48. 48 < 49 ⟹ Q ближе.
Почему можно: функция √ монотонно возрастает, поэтому порядок квадратов расстояний совпадает с порядком самих расстояний. (Тот самый приём «сравнивай квадраты» из § 1.1.4 — в коде он экономит вызов sqrt.)

**Ответ:** Q ближе (48 < 49); √ монотонен, порядок сохраняется.

### Г1.Ч1.З6

**Условие.** Найди координату x точки M(x, 0), равноудалённой от A(1, 3) и B(5, 1).

**Решение.** Приравниваем квадраты расстояний (корни не нужны — обе части неотрицательны):
(x−1)² + 3² = (x−5)² + 1²
x² − 2x + 1 + 9 = x² − 10x + 25 + 1
−2x + 10 = −10x + 26 ⟹ 8x = 16 ⟹ **x = 2**.
Проверка: d²(M,A) = 1 + 9 = 10; d²(M,B) = 9 + 1 = 10. ✓

**Ответ:** M(2, 0).

### Г1.Ч1.З7

**Условие.** Выведи формулу расстояния в пространстве из плоской формулы, применив теорему Пифагора дважды. Сопроводи вывод рисунком-схемой.

**Решение.** Пусть A(x₁,y₁,z₁), B(x₂,y₂,z₂), Δx, Δy, Δz — разности координат.

*Шаг 1 (горизонталь).* Спроецируем отрезок AB на плоскость Oxy: проекции точек — A′(x₁,y₁), B′(x₂,y₂). Длина проекции по плоской формуле (Пифагор №1): d_xy = √(Δx² + Δy²).

*Шаг 2 (вертикаль).* Отрезок AB — гипотенуза прямоугольного треугольника с катетами: горизонтальный d_xy (лежит в плоскости) и вертикальный |Δz| (перпендикулярен плоскости). Пифагор №2:
d² = d_xy² + Δz² = Δx² + Δy² + Δz² ⟹ d = √(Δx² + Δy² + Δz²). ∎

```
        B
        /|
       / | Δz          d² = d_xy² + Δz²
      /  |
     A---+   ← прямой угол
      \  |
   d_xy \|      (внизу — плоскость Oxy,
         B'      d_xy² = Δx² + Δy²)
```

**Ответ:** формула получается двукратным применением теоремы Пифагора — сначала в плоскости, затем в вертикальном треугольнике.

### Г1.Ч1.З8

**Условие.** Программист считает расстояние так: `d = abs(x2-x1) + abs(y2-y1)`. Тест «точки на одной горизонтали» проходит, а проверка радиуса взрыва работает неверно. Объясни, что именно он вычисляет, для каких пар точек его результат совпадает с настоящим расстоянием и насколько (максимум) он может ошибаться на плоскости.

**Решение.** Вычисляется **манхэттенское расстояние** |Δx| + |Δy| — длина пути «по сетке городских кварталов», а не прямого отрезка.
*Когда совпадает:* если движение только вдоль одной оси, то есть Δx = 0 или Δy = 0 (потому тест «на одной горизонтали» и проходил).
*Максимальная ошибка:* отношение manhattan/euclid = (|Δx|+|Δy|)/√(Δx²+Δy²) максимально при |Δx| = |Δy| = h: (2h)/(h√2) = **√2 ≈ 1.414** — завышение до ~41%. Взрыв в игре получает «квадратный» (повёрнутый ромбом) радиус вместо круглого.

**Ответ:** это манхэттенская метрика; совпадает при Δx = 0 или Δy = 0; завышает истинное расстояние максимум в √2 раза (на диагоналях).

### Г1.Ч1.З9

**Условие.** Экран 1920 × 1080, начало сверху-слева, y вниз. Переведи в систему «центр экрана, y вверх» пиксели: (960, 540), (0, 0), (1919, 1079). Куда попало начало старой системы?

**Решение.** Формулы из примера 4 главы: x′ = x − W/2 = x − 960; y′ = H/2 − y = 540 − y.
- (960, 540) → (0, 0) — центр экрана;
- (0, 0) → (−960, 540) — левый верхний угол: левее центра и выше;
- (1919, 1079) → (959, −539) — правый нижний.
Начало старой системы (0,0) — это точка (−960, 540) новой: левый верх.

**Ответ:** (0,0); (−960, 540); (959, −539); старое начало — в (−960, 540).

### Г1.Ч1.З10

**Условие.** Напиши пару функций: `pixelToCentered(px, py, w, h)` и обратную `centeredToPixel(cx, cy, w, h)`. Продемонстрируй на углах экрана 800×600, что композиция возвращает исходные значения.

**Решение (код выполнен, roundtrip проверен на всех углах и центре).**

```js
function pixelToCentered(px, py, w, h) {
  return [px - w / 2, h / 2 - py];   // сдвиг центра + переворот y
}
function centeredToPixel(cx, cy, w, h) {
  return [cx + w / 2, h / 2 - cy];   // обратный сдвиг; переворот y сам себе обратен
}
```

Демонстрация (800×600):

| пиксель | centered | обратно |
|---|---|---|
| (0, 0) | (−400, 300) | (0, 0) ✓ |
| (800, 0) | (400, 300) | (800, 0) ✓ |
| (0, 600) | (−400, −300) | (0, 600) ✓ |
| (800, 600) | (400, −300) | (800, 600) ✓ |
| (400, 300) | (0, 0) | (400, 300) ✓ |

Заметь: переворот y (`h/2 − py`) — инволюция (сам себе обратен), поэтому обе функции почти одинаковы.

### Г1.Ч1.З11

**Условие.** В GLSL доступны `gl_FragCoord.xy` (начало снизу-слева) и `u_resolution`. Запиши выражения: (а) `uv` в диапазоне [0,1]²; (б) координаты в диапазоне [−1,1]² с центром в середине экрана. В какой точке экрана (б) даст (0,0)?

**Решение.**

```glsl
vec2 uv = gl_FragCoord.xy / u_resolution;              // (а): [0,1]²
vec2 p  = 2.0 * gl_FragCoord.xy / u_resolution - 1.0;  // (б): [−1,1]²
```

(б) — это композиция (а) с аффинным «растяни вдвое и сдвинь»: p = 2·uv − 1. p = (0,0) когда uv = (0.5, 0.5), то есть `gl_FragCoord.xy = u_resolution / 2` — **геометрический центр экрана**. (Строго говоря, с точностью до половины пикселя: центры пикселей лежат на полуцелых координатах, § 1.1.5.)

**Ответ:** выражения выше; (0,0) — в центре экрана (пиксель с центром resolution/2).

## Часть 2. Ориентация и соглашения

### Г1.Ч2.З1

**Условие.** Для каждой конфигурации определи, правая система или левая: (а) x вправо, y вверх, z из экрана; (б) x вправо, y вверх, z в экран; (в) x влево, y вниз, z из экрана; (г) x вправо, z вверх, y от зрителя.

**Решение.** Тест правой руки: пальцы от x к y, большой палец — куда должен смотреть z у правой системы.
- (а) x→y даёт большой палец **из экрана** — совпадает с данным z: **правая** (это классика OpenGL).
- (б) z в экран — противоположно: **левая** (классика Direct3D).
- (в) относительно (а) перевёрнуты две оси (x и y) — чётное число флипов ориентацию сохраняет: **правая**.
- (г) x вправо, y от зрителя, z вверх: пальцы от x (вправо) к y (от зрителя), большой палец — вверх = z: **правая** (это соглашение Blender).

**Ответ:** (а) правая; (б) левая; (в) правая; (г) правая.

### Г1.Ч2.З2

**Условие.** В правой системе x направлен на восток, y — вертикально вверх. Куда направлен z?

**Решение.** Правило правой руки: пальцы от x (восток) сгибаются к y (вверх) — большой палец указывает **на юг**. Проверка координатами: в тройке восток-E, север-N, верх-U вычислим E × U по правилу «x̂ × ŷ = ẑ»-логики: z должен дополнять (x, y) до правой тройки, а юг = −север как раз даёт det > 0.

**Ответ:** на юг.

### Г1.Ч2.З3

**Условие.** Возьми правую систему и последовательно: флипни ось x; затем ось y; затем ось z. Какая ориентация после каждого шага? Сформулируй общее правило.

**Решение.** Каждый флип одной оси — зеркало, меняющее ориентацию:
правая →(флип x)→ **левая** →(флип y)→ **правая** →(флип z)→ **левая**.

**Ответ/правило:** ориентация меняется при каждом одиночном флипе; итог зависит только от **чётности** числа флипов — нечётное число меняет ориентацию, чётное сохраняет.

### Г1.Ч2.З4

**Условие.** Объясни, почему смену знака двух осей можно получить поворотом (укажи, каким реализуется (x, y, z) → (−x, −y, z)), а смену знака одной оси — нельзя.

**Решение.** (x, y, z) → (−x, −y, z) — это **поворот на 180° вокруг оси z**: каждая точка плоскости xy обходит пол-оборота, z не меняется. Проверка: поворот на 180° переводит (1,0,0) в (−1,0,0) и (0,1,0) в (0,−1,0) — совпадает.
Одиночный флип — зеркало: он превращает правую тройку осей в левую. Повороты — «непрерывные движения твёрдого тела»: вращая объект, нельзя превратить правую перчатку в левую. (Формальный язык появится в гл. 4: у поворотов det = +1, у зеркал det = −1, а det при композиции поворотов не может сменить знак.)

**Ответ:** двойной флип = поворот на 180° вокруг третьей оси; одиночный меняет ориентацию, что поворотам недоступно.

### Г1.Ч2.З5

**Условие.** Переведи точки (1, 2, 3), (0, 0, −5), (−2, 1, 0) из правой системы «y вверх, камера вдоль −z» в левую «y вверх, камера вдоль +z».

**Решение.** Обе системы согласны по x и y, различие — направление z: пересчёт (x, y, z) → (x, y, −z).

**Ответ:** (1, 2, −3); (0, 0, 5); (−2, 1, 0) (точка с z = 0 неподвижна).

### Г1.Ч2.З6

**Условие.** Художник передал модель из Blender (правая, z вверх): вершины (1, 2, 0), (0, −3, 4). Пересчитай их в координаты OpenGL (правая, y вверх). Куда переходит блендеровский «вверх» (0, 0, 1) и «вглубь сцены» (0, 1, 0)?

**Решение.** Правило из примера 6 главы: (x, y, z)_Blender → (x, z, −y)_OpenGL (поворот на −90° вокруг x — обе системы правые, зеркалить ничего не нужно).
- (1, 2, 0) → (1, 0, −2);
- (0, −3, 4) → (0, 4, 3);
- «вверх» (0, 0, 1) → (0, 1, 0) — верх OpenGL ✓;
- «вглубь» (0, 1, 0) → (0, 0, −1) — направление взгляда камеры OpenGL ✓ (обе проверки подтверждают правило).

**Ответ:** (1, 0, −2); (0, 4, 3); верх → (0,1,0), вглубь → (0,0,−1).

### Г1.Ч2.З7

**Условие.** На плоскости xy нарисован циферблат, стрелки идут по часовой (если смотреть со стороны +z в правой системе). Что произойдёт с направлением хода стрелок при отзеркаливании по оси x (x → −x)? А по обеим осям сразу? Как это связано с З3?

**Решение.** Флип x → −x — зеркало: круговое движение меняет направление, стрелки пойдут **против часовой** (как отражение часов в зеркале). Флип обеих осей = поворот на 180° (З4): картинка перевёрнута «вверх ногами», но стрелки идут **по-прежнему по часовой** — поворот листа с часами не меняет направления хода.
Связь с З3: направление обхода («по/против часовой») — и есть двумерная ориентация; одиночный флип её меняет, чётный — нет.

**Ответ:** по x — против часовой; по обеим — по часовой; чётность флипов = сохранение ориентации.

### Г1.Ч2.З8

**Условие.** Движок A: правая система, y вверх, камера смотрит вдоль −z. Движок B: левая, y вверх, камера вдоль +z. Напиши функцию `convertAtoB(x, y, z)` и обратную. Объясни, почему в этом случае они совпадают, и приведи пример пары соглашений, для которых прямая и обратная различались бы.

**Решение (код выполнен).**

```js
function convertAtoB(x, y, z) { return [x, y, -z]; }
function convertBtoA(x, y, z) { return [x, y, -z]; } // та же функция
```

Совпадают, потому что «сменить знак z» — **инволюция**: применённая дважды, она возвращает исходное ((x,y,−(−z)) = (x,y,z); проверено кодом на (7,−3,2)).
Пример несовпадающих: Blender → OpenGL из З6: прямая (x, y, z) → (x, z, −y), обратная — (x, y, z) → (x, −z, y). Применив прямую дважды, получим (x, −y, −z) ≠ исходному — не инволюция, потому обратная другая.

**Ответ:** обе функции — «flip z»; инволюция; контрпример — пара соглашений с перестановкой осей (Blender↔OpenGL).

## Часть 3. Точки и направления

### Г1.Ч3.З1

**Условие.** Классифицируй как точку или вектор: позиция врага; скорость снаряда; направление ветра; координаты вершины меша; смещение камеры за кадр; положение курсора; нормаль к поверхности.

**Решение.** Критерий: «где» — точка; «куда/насколько» (не привязано к месту) — вектор.

**Ответ:** позиция врага — точка; скорость — вектор; направление ветра — вектор; вершина меша — точка; смещение камеры — вектор; курсор — точка; нормаль — вектор.

### Г1.Ч3.З2

**Условие.** Даны P(3, −1, 2) и Q(0, 4, 2). Найди вектор из P в Q и вектор из Q в P. Как они связаны?

**Решение.** «Куда минус откуда»: PQ = Q − P = (0−3, 4−(−1), 2−2) = (−3, 5, 0); QP = P − Q = (3, −5, 0).

**Ответ:** (−3, 5, 0) и (3, −5, 0) — противоположные векторы (QP = −PQ): одна и та же стрелка, пройденная в обратную сторону.

### Г1.Ч3.З3

**Условие.** Точка A(5, 5, 5), смещение v = (−2, 0, 7). Найди A + v. Затем найди точку B такую, что B + v = A.

**Решение.** A + v = (3, 5, 12). Из B + v = A: B = A − v = (5−(−2), 5−0, 5−7) = (7, 5, −2). Проверка: (7,5,−2) + (−2,0,7) = (5,5,5) ✓.

**Ответ:** A + v = (3, 5, 12); B = (7, 5, −2).

### Г1.Ч3.З4

**Условие.** Персонаж стартует в S(0, 0, 0) и получает последовательно смещения (2, 0, 1), (−1, 0, 3), (0, 5, −4). Где он окажется? Изменится ли ответ при другом порядке смещений — и почему?

**Решение.** Конечная точка = S + сумма смещений = (2−1+0, 0+0+5, 1+3−4) = **(1, 5, 0)**.
Порядок не важен: сложение векторов коммутативно и ассоциативно (u + v = v + u), итоговая точка зависит только от суммарного смещения. (Промежуточные точки маршрута, конечно, различаются.)

**Ответ:** (1, 5, 0); не изменится — сложение векторов коммутативно.

### Г1.Ч3.З5

**Условие.** Найди середину отрезка с концами P(2, 8, −4) и Q(6, 0, 2). Запиши вычисление двумя способами: как (P+Q)/2 и как P + (Q−P)/2 — и объясни, почему второй способ «легален» с точки зрения таблицы операций, а первый требует оговорки о весах.

**Решение.**
*Способ 1:* (P+Q)/2 = ((2+6)/2, (8+0)/2, (−4+2)/2) = (4, 4, −1).
*Способ 2:* Q − P = (4, −8, 6) — вектор (легально); его половина (2, −4, 3) — вектор; P + (2, −4, 3) = (4, 4, −1) — точка + вектор = точка (легально). Все шаги — из таблицы § 1.3.2.
*Почему первый требует оговорки:* «P + Q» — сумма точек, сама по себе не определена; выражение (P+Q)/2 осмысленно только как **взвешенная комбинация с суммой весов 1** (0.5 + 0.5 = 1) — тогда оно не зависит от выбора начала координат. Раскрыв скобки, способ 1 сводится ко второму: P + (Q−P)/2.

**Ответ:** (4, 4, −1); второй способ использует только определённые операции, первый — аффинная комбинация, легален лишь благодаря сумме весов 1.

### Г1.Ч3.З6

**Условие.** Найди ошибку в псевдокоде: `lookDir = camera.position + target.position`. Что здесь вычислено, почему это не имеет смысла, и как правильно?

**Решение.** Сложены две **точки** — операция без геометрического смысла: результат зависит от того, где стоит начало координат (перенеси мир — «сумма» уедет по-другому, чем обе точки). Направление взгляда — это смещение от камеры к цели: разность точек.

**Ответ:** правильно `lookDir = target.position - camera.position` («куда минус откуда»); при необходимости нормализовать (гл. 2).

### Г1.Ч3.З7

**Условие.** Камера в C(0, 2, 10), цель в T(4, 2, 7). Найди вектор направления взгляда и дистанцию до цели. Объясни, как эта задача связана с § 1.1.4.

**Решение.** Направление: T − C = (4, 0, −3). Дистанция: ‖T − C‖ = √(16 + 0 + 9) = **5**.
Связь: формула расстояния § 1.1.4 — это в точности длина вектора-разности двух точек; «расстояние между точками» и «длина смещения между ними» — одно и то же число.

**Ответ:** направление (4, 0, −3), дистанция 5.

### Г1.Ч3.З8

**Условие.** *(забегание вперёд — вернёмся в главе 7)* Точку кодируют как (x, y, z, 1), вектор — как (x, y, z, 0). Перенос на (tx, ty, tz) действует правилом (x, y, z, w) → (x + tx·w, y + ty·w, z + tz·w, w). Проверь, что происходит с точкой (1, 2, 3, 1) и вектором (1, 2, 3, 0) при переносе на (10, 20, 30). Почему такое поведение — именно то, что нужно?

**Решение.**
*Точка* (w = 1): (1 + 10·1, 2 + 20·1, 3 + 30·1, 1) = (11, 22, 33, 1) — сдвинулась на весь вектор переноса.
*Вектор* (w = 0): (1 + 10·0, 2 + 20·0, 3 + 30·0, 0) = (1, 2, 3, 0) — не изменился вовсе.
Почему это правильно: точка — «где» — обязана переехать вместе со сценой; вектор — «куда» (направление ветра, взгляд, нормаль) — не привязан к месту, перенос сцены не должен его трогать. Одна и та же формула обслуживает оба случая без единого условия — w работает переключателем. (В главе 7 это станет матрицей 4×4.)

**Ответ:** точка → (11, 22, 33, 1), вектор не изменился; w автоматически отличает «переносимое» от «непереносимого».
