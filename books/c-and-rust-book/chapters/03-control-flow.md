# Глава 3. Управляющие конструкции

## Введение

К концу главы 2 наша программа умела хранить данные в переменных, вычислять над ними выражения и обмениваться значениями с пользователем через `printf`/`scanf`. Но у неё оставался фундаментальный изъян: она выполнялась **строго сверху вниз, одинаково при любых входных данных**. Прочитать число и сообщить, чётное оно или нечётное; повторить действие сто раз; отвергнуть неверный ввод — всё это пока недостижимо.

Эта глава снимает ограничение. Мы научимся двум вещам, из которых складывается вся логика программ:

- **ветвлению** — выбирать, какой участок кода выполнить, в зависимости от условия (`if`/`else`, `switch`, тернарный оператор);
- **повторению** — выполнять участок кода много раз (`while`, `for`, `do-while`), управляя повторами через `break` и `continue`.

Оба механизма опираются на понятие из §2.2.2: **условие — это выражение**, дающее «истину» или «ложь». Мы уточним, что именно C считает истиной, и научимся строить из простых сравнений сложную логику.

После изучения главы вы будете уметь:

- писать ветвления любой сложности: одиночные `if`, цепочки `else if`, вложенные условия, множественный выбор через `switch`;
- выбирать подходящий цикл (`while`, `for`, `do-while`) под задачу и управлять его ходом (`break`, `continue`);
- распознавать и не допускать классические ошибки — бесконечный цикл, «off-by-one», забытый `break`, висячий `else`, `=` вместо `==`;
- применять базовые алгоритмические паттерны: счётчик, накопитель, флаг, поиск, чтение потока чисел до конца ввода.

> Как и в главе 2, весь код здесь — прямая последовательность внутри `main`: своих функций (глава 4), массивов (глава 5) и указателей (глава 6) ещё нет. Поэтому обрабатывать *много* значений мы будем, не сохраняя их, а «пропуская через» переменные-накопители (сумма, счётчик, максимум) — это ровно то, для чего и нужны циклы. Ввод по-прежнему читаем через `scanf` с `&` (§2.3.2).

## Часть 3.1. Ветвления

### §3.1.1. Оператор `if`/`else` и истинность в C

**Определения.**

- **Оператор `if`** выполняет вложенную инструкцию, только если условие истинно:

  ```c
  if (условие)
      инструкция;
  ```

- **Форма `if`/`else`** задаёт две ветви — на случай истины и на случай лжи; выполняется ровно одна:

  ```c
  if (условие)
      инструкция1;   // если условие истинно
  else
      инструкция2;   // если условие ложно
  ```

- **Условие** — любое выражение (§2.2). C считает его **ложным**, если оно равно `0`, и **истинным** при **любом ненулевом** значении. Специального типа «булев» в базовом C нет: истинность — это «ноль/не ноль». Поэтому и сравнения из §2.2.2 (дающие ровно 0 или 1), и арифметические выражения (`n % 2`, `x - 5`), и просто переменная (`count`) — всё это допустимые условия.
- **Блок** (составная инструкция) — несколько инструкций в фигурных скобках `{ … }`, воспринимаемых как одна. Если под `if` или `else` должно выполниться больше одной инструкции, их объединяют в блок.

**Интуиция.** `if` — это развилка на пути исполнения: программа доходит до условия, вычисляет его и сворачивает на одну из двух дорог. После развилки дороги снова сходятся, и выполнение продолжается за `if`.

```
        вычислить условие
              │
        ┌─────┴─────┐
   истина│           │ложь
        ▼           ▼
   инструкция1   инструкция2      (ветви else может и не быть)
        └─────┬─────┘
              ▼
     продолжение программы
```

**Разобранные примеры.**

*Пример 1. Чётное или нечётное.* Условие — сравнение остатка с нулём (`%` из §2.2.1):

```c
#include <stdio.h>

int main(void)
{
    int n = 7;
    if (n % 2 == 0)
        printf("%d is even\n", n);
    else
        printf("%d is odd\n", n);
    return 0;
}
```

Вывод:

```
7 is odd
```

`7 % 2` равно `1`; сравнение `1 == 0` даёт `0` (ложь), поэтому выполняется ветвь `else`.

*Пример 2. Истинность — это «ноль/не ноль».* Условием может быть само значение, без явного сравнения:

```c
#include <stdio.h>

int main(void)
{
    int x = 42;
    if (x)                       // x != 0  ->  истина
        printf("x is nonzero (truthy)\n");
    int y = 0;
    if (y)
        printf("this line never runs\n");
    else
        printf("y is zero (falsy)\n");
    return 0;
}
```

Вывод:

```
x is nonzero (truthy)
y is zero (falsy)
```

`if (x)` при `x = 42` истинно (число ненулевое), `if (y)` при `y = 0` ложно. Запись `if (x)` — это краткая и идиоматичная форма `if (x != 0)`.

*Пример 3. Блок под `if`.* Чтобы под условием выполнилось несколько строк, их берут в `{ }`:

```c
#include <stdio.h>

int main(void)
{
    int temp = 30;
    if (temp > 25) {
        printf("It is warm.\n");
        printf("Consider a t-shirt.\n");
    }
    printf("Done.\n");
    return 0;
}
```

Вывод:

```
It is warm.
Consider a t-shirt.
Done.
```

Обе строки внутри `{ }` относятся к `if`, а `printf("Done.\n")` — уже вне его и печатается всегда.

**Связь с практикой.** Ветвление — первый шаг к «умной» программе: проверить, что ввод корректен (вспомните возврат `scanf` из §2.3.2 — теперь мы можем на него *среагировать*), что делитель не ноль (§2.2.1), что значение в допустимом диапазоне. Привычка **всегда ставить фигурные скобки**, даже под одной инструкцией, — дешёвая страховка от целого класса ошибок (см. подводные камни и §3.1.2).

**Типичные ошибки и подводные камни.**

- **`=` вместо `==` в условии.** `if (x = 5)` — это *присваивание*: оно кладёт 5 в `x` и даёт значение 5 (истину), поэтому ветвь выполнится **всегда**, а `x` молча испортится. Нужно `if (x == 5)`. Компилятор с `-Wall` предупреждает (`-Wparentheses`) — не игнорируйте. (Приём «сравнивать константу слева»: `if (5 == x)` превратит опечатку в ошибку компиляции.)
- **Точка с запятой сразу после `if`.** `if (x > 0);` — тело `if` пусто (одна «пустая инструкция» `;`), а следующая строка выполнится безусловно. Ошибка тихая: компилятор вправе не возражать.
- **Отсутствие блока при нескольких строках.** Под `if` без `{ }` попадает только *одна* следующая инструкция; остальные выполняются всегда, как бы вы их ни отступили (см. §3.1.2, «висячий else»). Отступы на исполнение не влияют.
- **Сравнение вещественных через `==`** в условии ненадёжно (§2.1.3): `if (x == 0.3)` может оказаться ложным.

**Источники:** C2 (гл. 5), C1 (гл. 3); нормативно об истинности условия и операторе `if` — C5; дидактика ветвлений — C7.

### §3.1.2. Цепочки условий, вложенность и «висячий `else`»

**Определения.**

- **Цепочка `else if`** выбирает *одну* ветвь из многих, проверяя условия по порядку сверху вниз; как только одно истинно, его ветвь выполняется, а остальные пропускаются:

  ```c
  if (условие1)      ветвь1;
  else if (условие2) ветвь2;
  else if (условие3) ветвь3;
  else               ветвь_иначе;   // необязательная
  ```

  Синтаксически это не отдельная конструкция, а вложенные `if` в ветвях `else`; читается как «лестница».
- **Вложенное ветвление** — `if` внутри ветви другого `if`: так проверяют условия, зависящие друг от друга.
- **Правило связывания `else`** (важное!): `else` всегда относится к **ближайшему предшествующему `if`**, у которого ещё нет своего `else`, — независимо от отступов. Отсюда ловушка «висячего else» (dangling else).

**Интуиция.** Цепочка `else if` — это ряд сит: значение просеивается сверху вниз и «застревает» в первом сите, чьё условие выполнилось. Порядок сит важен: более узкие (частные) условия должны идти раньше более широких, иначе широкое перехватит значение первым.

```
score >= 90 ? ──да──▶ 'A'
    │нет
score >= 80 ? ──да──▶ 'B'
    │нет
score >= 70 ? ──да──▶ 'C'
    │нет
   ...
    │нет
          └────────▶ 'F'   (else — «всё остальное»)
```

**Разобранные примеры.**

*Пример 1. Оценка по баллу (цепочка `else if`).*

```c
#include <stdio.h>

int main(void)
{
    int score = 83;
    char grade;
    if (score >= 90)
        grade = 'A';
    else if (score >= 80)
        grade = 'B';
    else if (score >= 70)
        grade = 'C';
    else if (score >= 60)
        grade = 'D';
    else
        grade = 'F';
    printf("score %d -> grade %c\n", score, grade);
    return 0;
}
```

Вывод:

```
score 83 -> grade B
```

`83 >= 90` ложно, `83 >= 80` истинно → `grade = 'B'`, дальше цепочка не проверяется. Порядок условий гарантирует, что «81» — это `B`, а не `D`: до `>= 60` дело не доходит.

*Пример 2. Вложенные условия.* Классифицируем число сразу по двум признакам — знаку и чётности:

```c
#include <stdio.h>

int main(void)
{
    int n = -6;
    if (n >= 0) {
        printf("nonnegative, ");
        if (n % 2 == 0) printf("even\n");
        else            printf("odd\n");
    } else {
        printf("negative, ");
        if (n % 2 == 0) printf("even\n");
        else            printf("odd\n");
    }
    return 0;
}
```

Вывод:

```
negative, even
```

Внешний `if` выбрал ветвь «negative» (`-6 >= 0` ложно), внутренний — «even» (`-6 % 2` равно 0, знак остатка — знак делимого, §2.2.1).

*Пример 3. Висячий `else`.* Отступы обманчивы: `else` относится к **внутреннему** `if`, а не к внешнему, хотя выровнен под внешний:

```c
#include <stdio.h>

int main(void)
{
    int a = 1, b = -5;
    if (a > 0)
        if (b > 0)
            printf("both positive\n");
        else
            printf("a>0 but b<=0 (else pairs with INNER if)\n");
    return 0;
}
```

Вывод:

```
a>0 but b<=0 (else pairs with INNER if)
```

Здесь `a > 0` истинно, `b > 0` ложно, поэтому срабатывает `else` внутреннего `if`. При сборке `gcc -Wall` честно предупреждает:

```
warning: suggest explicit braces to avoid ambiguous 'else' [-Wdangling-else]
```

Правило спасает от неоднозначности, но код всё равно стоит писать так, чтобы намерение не приходилось угадывать (см. подводные камни).

**Связь с практикой.** Цепочки `else if` — стандартный способ разложить величину по категориям (оценка, уровень заряда, диапазон температур). Вложенные `if` появляются, когда одна проверка имеет смысл только внутри другой (например, «файл открылся → *и* в нём есть данные» — это уже главы 7 и 10). Главный практический вывод из «висячего else»: **фигурные скобки у вложенных `if` — не роскошь, а способ сделать связывание однозначным для человека**.

**Типичные ошибки и подводные камни.**

- **Неверный порядок в цепочке.** Если поставить широкое условие раньше узкого (`if (score >= 60) … else if (score >= 90) …`), ветвь `>= 90` станет недостижимой: `95` перехватит первое же `>= 60`. Более частные условия — выше.
- **Висячий `else`.** Всегда обрамляйте вложенные `if` блоками `{ }` — тогда `else` привязан однозначно, а отступы совпадают со смыслом. Реагируйте на `-Wdangling-else`.
- **Забытый `else` (или лишний `if`).** Цепочка `if … if …` (без `else`) — это *несколько независимых* проверок: может сработать больше одной. `if … else if …` — это *выбор одной*. Путать их — логическая ошибка.
- **Перекрывающиеся диапазоны.** В `if (x > 0) … else if (x > 100) …` вторая ветвь мертва. Следите, чтобы условия делили пространство значений без «дыр» и без бесполезных перекрытий.

**Источники:** C2 (гл. 5), C1 (гл. 3); нормативно о связывании `else` с ближайшим `if` — C5; C7.

### §3.1.3. Оператор `switch`

**Определения.**

- **`switch`** выбирает ветвь по значению **целочисленного** выражения, сравнивая его с набором констант-меток `case`:

  ```c
  switch (выражение) {
      case константа1:
          инструкции;
          break;
      case константа2:
          инструкции;
          break;
      default:            // необязательная ветвь «ни одна из меток»
          инструкции;
          break;
  }
  ```

- **`case константа:`** — метка перехода. Константа должна быть целочисленной и известной на этапе компиляции (`char` подходит — это целое, §2.1.3; переменная или диапазон — нет).
- **`break`** внутри `switch` немедленно выходит из него. **Без `break` управление «проваливается» (fallthrough)** в следующий `case` — выполнение продолжается по порядку, игнорируя метки, пока не встретит `break` или конец `switch`.
- **`default`** выполняется, если не совпала ни одна метка. Её можно опустить, но лучше предусмотреть.

**Интуиция.** `switch` — это не «цепочка сравнений», а **вход по метке**: вычисленное значение указывает, *с какого места* начать выполнение, а дальше код идёт подряд вниз до `break`. Метки `case` — это двери в общий коридор; `break` — выход из коридора. Если не выйти, пойдёшь через следующую дверь изнутри.

```
switch (choice)
   ├─ case 1: ─▶ ... ─ break ─┐
   ├─ case 2: ─▶ ... ─ break ─┤
   ├─ case 3: ─▶ ... ─ break ─┤
   └─ default:─▶ ... ─ break ─┘
                               └──▶ после switch
   (нет break  ⇒  «проваливание» в следующий case)
```

**Разобранные примеры.**

*Пример 1. Меню по числу.*

```c
#include <stdio.h>

int main(void)
{
    int choice = 2;
    switch (choice) {
        case 1:
            printf("You chose ONE\n");
            break;
        case 2:
            printf("You chose TWO\n");
            break;
        case 3:
            printf("You chose THREE\n");
            break;
        default:
            printf("Unknown choice\n");
            break;
    }
    return 0;
}
```

Вывод:

```
You chose TWO
```

Значение `2` совпало с `case 2`; после его `printf` сработал `break` — остальные ветви пропущены.

*Пример 2. Намеренное «проваливание» для группировки.* Несколько меток подряд без кода между ними — законный приём: одна ветвь обслуживает целую группу значений. Проверим, гласная ли буква:

```c
#include <stdio.h>

int main(void)
{
    char c = 'e';
    switch (c) {
        case 'a':
        case 'e':
        case 'i':
        case 'o':
        case 'u':
            printf("%c is a vowel\n", c);
            break;
        default:
            printf("%c is a consonant\n", c);
    }
    return 0;
}
```

Вывод:

```
e is a vowel
```

Метка `case 'e'` не имеет своего тела и «проваливается» до общего кода под `case 'u'` — так пять гласных обрабатываются одной ветвью.

*Пример 3. Забытый `break` — случайное «проваливание».* А вот та же механика уже как *ошибка*:

```c
#include <stdio.h>

int main(void)
{
    int n = 1;
    switch (n) {
        case 1:
            printf("one\n");
            // забыт break! управление "проваливается" в case 2
        case 2:
            printf("two\n");
            break;
        case 3:
            printf("three\n");
            break;
    }
    return 0;
}
```

Вывод:

```
one
two
```

Совпал `case 1`, но из-за отсутствующего `break` выполнение продолжилось в `case 2` и напечатало лишнюю строку. Это настолько частая ошибка, что `gcc -Wall` предупреждает о ней:

```
warning: this statement may fall through [-Wimplicit-fallthrough=]
```

Намеренное «проваливание» (как в примере 2) компилятор не ругает, потому что там между метками нет кода; опасно именно «проваливание» через непустую ветвь.

**Связь с практикой.** `switch` выразительнее длинной цепочки `else if`, когда выбор идёт по *конкретным значениям* одной целой величины: пункт меню, код команды, символ операции. Он и читается яснее (все варианты выстроены в столбец), и часто компилируется в эффективный переход по таблице. Для *диапазонов* (`score >= 90`) `switch` не годится — там остаётся цепочка `else if`.

**Типичные ошибки и подводные камни.**

- **Забытый `break`** — управление проваливается в следующую ветвь (пример 3). Ставьте `break` в конце каждой ветви; реагируйте на `-Wimplicit-fallthrough`.
- **`case` с переменной или диапазоном.** `case x:` или `case 1 ... 10:` в стандартном C недопустимы: метка — только целочисленная константа. Диапазоны и условия по переменным — это `if`/`else if`.
- **Нет `default`.** Если не совпала ни одна метка и `default` отсутствует, `switch` не делает ничего — легко упустить некорректный ввод. Предусматривайте `default`.
- **Объявление переменной прямо в `switch` без блока.** Объявление с инициализацией сразу после `case` (в обход `{ }`) вызывает ошибку компиляции — оборачивайте тело ветви в `{ }`, если объявляете в нём переменные.
- **`switch` по вещественному.** Работает только с целочисленным выражением; `double` в `switch` недопустим.

**Источники:** C2 (гл. 5), C1 (гл. 3); нормативно о `switch`, `case`, `default` и «проваливании» — C5; поведение и предупреждения — C6.

### §3.1.4. Тернарный (условный) оператор `?:`

**Определения.**

- **Условный оператор** `условие ? выражение1 : выражение2` — единственный **тернарный** (трёхоперандный) оператор C. Он вычисляет `условие`; если истинно — результатом всего оператора становится `выражение1`, иначе — `выражение2`. Вычисляется **только одна** из двух ветвей.
- В отличие от `if`, это **выражение**, а не инструкция: у него есть значение, которое можно присвоить, передать в `printf`, использовать в арифметике.

**Интуиция.** `?:` — это «`if`/`else`, ужатый до значения». Там, где `if` *делает* (выполняет инструкцию), `?:` *возвращает* (даёт значение). Он удобен для коротких выборов «одно из двух значений», но быстро становится нечитаемым, если в него впихивать логику.

```
(a > b) ? a : b
   │      │   └── значение, если условие ложно
   │      └────── значение, если условие истинно
   └───────────── условие (как в if)
```

**Разобранные примеры.**

*Пример 1. Максимум из двух.*

```c
#include <stdio.h>

int main(void)
{
    int a = 3, b = 8;
    int max = (a > b) ? a : b;
    printf("max = %d\n", max);
    return 0;
}
```

Вывод:

```
max = 8
```

`a > b` (3 > 8) ложно, поэтому результат — `b`, то есть 8. Эквивалентный `if` занял бы четыре строки; здесь выбор уместился в одно присваивание.

*Пример 2. Вложенный тернарный оператор — знак числа.* Правая ветвь сама может быть тернарным оператором (он правоассоциативен, §2.2.4):

```c
#include <stdio.h>

int main(void)
{
    int n = -4;
    int sign = (n > 0) ? 1 : (n < 0) ? -1 : 0;
    printf("sign(%d) = %d\n", n, sign);
    return 0;
}
```

Вывод:

```
sign(-4) = -1
```

Читается как лестница: `n > 0` ? `1` : (`n < 0` ? `-1` : `0`). Для `-4` первое условие ложно, второе истинно → `-1`.

*Пример 3. Тернарный оператор внутри выражения.* Результат можно сразу использовать — например, выбрать символ для печати:

```c
#include <stdio.h>

int main(void)
{
    int n = 7;
    char parity = (n % 2 == 0) ? 'E' : 'O';
    printf("%d -> %c\n", n, parity);
    return 0;
}
```

Вывод:

```
7 -> O
```

`7 % 2 == 0` ложно → выбран символ `'O'`.

**Связь с практикой.** Тернарный оператор ценен для коротких присваиваний «выбери одно из двух» и для подстановки значения прямо в вызов (`printf`, вычисление). Он делает код компактнее там, где полноценный `if`/`else` был бы многословен. Но злоупотребление им — прямой путь к нечитаемому коду: если в ветвях не значения, а действия, или условий больше двух, берите обычный `if`.

**Типичные ошибки и подводные камни.**

- **Логика вместо значения.** `?:` предназначен возвращать *значение*, а не выполнять действия. Не пишите `(x > 0) ? printf("+") : printf("-");` — здесь уместен `if`.
- **Глубокая вложенность.** Три и более вложенных `?:` почти нечитаемы; замените цепочкой `else if`.
- **Низкий приоритет.** `?:` связывает слабее арифметики: `a > b ? a : b + 1` — это `(a > b) ? a : (b + 1)`, а не `((a > b) ? a : b) + 1`. Сомневаетесь — ставьте скобки (§2.2.4).
- **Разные типы ветвей.** `условие ? 1 : 2.5` приведёт обе ветви к общему типу (`double`) по правилам §2.2.5; следите за типом результата, если он важен.

**Источники:** C2 (гл. 4–5), C1 (гл. 2–3); нормативно об условном операторе и вычислении одной ветви — C5.

## Часть 3.2. Циклы

### §3.2.1. Цикл `while` и анатомия повторения

**Определения.**

- **Цикл `while`** повторяет тело, **пока** условие истинно:

  ```c
  while (условие)
      тело;          // обычно блок { ... }
  ```

  Условие проверяется **перед** каждой итерацией (в том числе перед первой). Если оно ложно сразу, тело не выполнится ни разу.
- **Итерация** — одно выполнение тела цикла.
- У любого «правильного» цикла есть три части, даже когда они разбросаны по коду: **инициализация** (задать стартовое состояние), **условие продолжения** (когда повторять) и **изменение состояния** в теле (шаг к завершению). Если тело не приближает условие к ложности, цикл **бесконечен**.

**Интуиция.** `while` — это «повторяй, пока держится условие». Представьте контрольную точку перед входом: на ней проверяют пропуск (условие). Есть пропуск — заходишь на круг (тело), в конце круга что-то меняется, и ты снова у контрольной точки. Нет пропуска — уходишь (цикл закончился).

```
      ┌───────────────┐
      ▼               │
  проверить условие   │
      │               │
 истина│  ложь        │
      ▼    └──▶ выход  │
    тело ──▶ шаг ──────┘
```

**Разобранные примеры.**

*Пример 1. Обратный отсчёт.* Счётчик уменьшается в теле — это и есть «шаг к завершению»:

```c
#include <stdio.h>

int main(void)
{
    int i = 5;                 // инициализация
    while (i > 0) {            // условие продолжения
        printf("%d ", i);
        i--;                   // шаг: приближаемся к i == 0
    }
    printf("\nLiftoff!\n");
    return 0;
}
```

Вывод:

```
5 4 3 2 1 
Liftoff!
```

Тело выполнилось для `i` = 5, 4, 3, 2, 1; при `i = 0` условие `0 > 0` ложно — цикл завершился. Уберите `i--` — и условие никогда не станет ложным: получится бесконечный цикл (см. подводные камни).

*Пример 2. Сумма 1…n (накопитель).* Переменная `sum` накапливает результат по одному слагаемому за итерацию:

```c
#include <stdio.h>

int main(void)
{
    int n = 5;
    int sum = 0;               // накопитель стартует с нейтрального значения
    int i = 1;
    while (i <= n) {
        sum += i;              // накопить
        i++;                   // шаг
    }
    printf("sum 1..%d = %d\n", n, sum);
    return 0;
}
```

Вывод:

```
sum 1..5 = 15
```

За пять итераций `sum` прошёл значения 1, 3, 6, 10, 15. Это фундаментальный **паттерн накопителя**: до цикла — нейтральное начальное значение (0 для суммы, 1 для произведения), в теле — обновление.

*Пример 3. Условие проверяется до тела.* Если оно ложно с самого начала, тело не выполнится ни разу:

```c
#include <stdio.h>

int main(void)
{
    int count = 0;
    int i = 10;
    while (i < 5) {            // условие ложно с самого начала
        count++;
        i++;
    }
    printf("body executed %d times\n", count);
    return 0;
}
```

Вывод:

```
body executed 0 times
```

`10 < 5` ложно сразу, поэтому `while` пропускает тело целиком. Это ключевое отличие `while` от `do-while` (§3.2.3), где тело всегда выполняется хотя бы раз.

**Связь с практикой.** `while` — выбор, когда число повторений заранее неизвестно и определяется *условием*: «пока есть ввод», «пока не сошлось», «пока пользователь не ввёл корректное значение». Дисциплина «инициализация → условие → шаг» — чек-лист против самой частой ошибки новичка (бесконечный цикл): прежде чем запускать, спросите себя, *что* в теле однажды сделает условие ложным.

**Типичные ошибки и подводные камни.**

- **Бесконечный цикл.** Если тело не меняет переменных условия (забыт `i--`/`i++`), цикл не кончится. Признак при отладке — программа «висит» и ничего не выводит (из-за буферизации, §2.3.3, уже напечатанное может не появиться). Прерывание — `Ctrl+C`.
- **`;` сразу после `while (...)`.** `while (i < n);` — тело пусто, а настоящий «блок» ниже выполнится один раз. Если условие при этом истинно и неизменно — зависание.
- **Изменение переменной не в ту сторону.** `while (i > 0) { …; i++; }` уводит `i` от нуля — снова бесконечность. Убедитесь, что шаг ведёт к ложности условия.
- **Off-by-one.** `i < n` и `i <= n` дают разное число итераций. Точно решите, включать ли границу, и проверьте на маленьком `n` вручную (подробнее — §3.2.2).

**Источники:** C2 (гл. 6), C1 (гл. 3); нормативно о `while` и проверке условия перед итерацией — C5; C7.

### §3.2.2. Цикл `for`

**Определения.**

- **Цикл `for`** собирает три части управления в одну строку:

  ```c
  for (инициализация; условие; изменение)
      тело;
  ```

  Порядок работы: **инициализация** выполняется один раз до цикла; затем перед каждой итерацией проверяется **условие**; после каждой итерации выполняется **изменение**. То есть `for` — это компактная запись эквивалентного `while`.
- Счётчик можно **объявить прямо в заголовке**: `for (int i = 0; …)`. Такая переменная видна только внутри цикла (её **область видимости** — сам `for`) и исчезает после него. Это возможность C99 и новее (мы собираем стандартом C17).
- Любая из трёх частей может быть пустой: `for (;;)` — «вечный» цикл (условие-«пусто» считается истинным).

**Интуиция.** `for` и `while` — одно и то же, разложенное иначе. `for` **собирает** управление циклом в заголовок, поэтому удобен, когда число повторений известно или задаётся счётчиком. `while` **оставляет** управление в теле — удобен, когда условие «естественное», а явного счётчика нет.

```
for (int i = 0; i < n; i++)   эквивалентно   { int i = 0;
    тело;                                        while (i < n) {
                                                     тело;
                                                     i++;
                                                 } }
   └──── i живёт только внутри for ────┘         (i виден и после while)
```

**Разобранные примеры.**

*Пример 1. Сумма счётчиком.* Тот же результат, что у `while`-примера §3.2.1, но компактнее:

```c
#include <stdio.h>

int main(void)
{
    int sum = 0;
    for (int i = 1; i <= 5; i++)
        sum += i;
    printf("sum = %d\n", sum);
    return 0;
}
```

Вывод:

```
sum = 15
```

Инициализация `int i = 1`, условие `i <= 5`, изменение `i++` стоят рядом — весь «мотор» цикла виден в одной строке.

*Пример 2. Факториал (накопитель-произведение).* Начальное значение накопителя — 1 (нейтральное для умножения); тип `long long`, потому что факториалы растут быстро (§2.1.2):

```c
#include <stdio.h>

int main(void)
{
    int n = 5;
    long long fact = 1;
    for (int i = 2; i <= n; i++)
        fact *= i;
    printf("%d! = %lld\n", n, fact);
    return 0;
}
```

Вывод:

```
5! = 120
```

`fact` проходит значения 2, 6, 24, 120. Начни мы с `fact = 0`, любое умножение дало бы 0 — вот зачем произведению нейтральная единица.

*Пример 3. Шаг больше единицы.* Часть «изменение» — любое выражение, не обязательно `++`:

```c
#include <stdio.h>

int main(void)
{
    for (int k = 0; k <= 10; k += 2)
        printf("%d ", k);
    printf("\n");
    return 0;
}
```

Вывод:

```
0 2 4 6 8 10 
```

`k += 2` (§2.2.3) даёт чётные числа. Так же легко идти вниз (`k--`), через 5 (`k += 5`) и т. д.

**Связь с практикой.** `for` — рабочая лошадка «сделать ровно N раз» и обхода диапазонов. Когда в главе 5 появятся массивы, именно `for (int i = 0; i < n; i++)` станет каноническим способом их перебрать. Объявление счётчика в заголовке — хорошая привычка: переменная не «утекает» за пределы цикла и не конфликтует с другими.

**Типичные ошибки и подводные камни.**

- **Off-by-one (ошибка на единицу).** `i < n` даёт `n` итераций (0…n−1), `i <= n` — `n+1` (0…n). Перепутать — значит обработать на один элемент меньше или больше. Проверяйте на маленьком `n`: при `n = 3` цикл `for (i = 0; i < 3; i++)` должен дать `i` = 0, 1, 2.
- **`;` после заголовка `for`.** `for (int i = 0; i < n; i++);` — тело пусто; следующий блок выполнится один раз. Тихая ошибка.
- **Изменение счётчика и в заголовке, и в теле.** Если менять `i` ещё и внутри тела, легко получить не то число итераций или бесконечность. Пусть шагом управляет только часть «изменение».
- **Использование счётчика после цикла.** Если `i` объявлен в заголовке (`for (int i …)`), после цикла его уже нет — это по замыслу. Нужен итог — заведите отдельную переменную вне цикла.
- **Вещественный счётчик.** `for (double x = 0; x != 1.0; x += 0.1)` может не завершиться: `0.1` неточен (§2.1.3), и `x` «промахнётся» мимо `1.0`. Для дробных шагов считайте целыми и делите при использовании.

**Источники:** C2 (гл. 6), C1 (гл. 3); нормативно о `for`, порядке частей и области видимости счётчика — C5; C7.

### §3.2.3. Цикл `do-while` и выбор конструкции

**Определения.**

- **Цикл `do-while`** выполняет тело, а условие проверяет **после** итерации:

  ```c
  do {
      тело;
  } while (условие);      // не забудьте точку с запятой!
  ```

  Поэтому тело выполняется **хотя бы один раз**, даже если условие ложно с самого начала. Это единственное отличие от `while`.
- **Три цикла — один смысл, разные удобства.** `while` и `for` проверяют условие *до* тела (могут не выполнить его ни разу); `do-while` — *после* (выполнит минимум один раз). Любой из них выражается через другой; выбор — вопрос читаемости.

**Интуиция.** `do-while` — «сделай, потом посмотри, повторять ли». Классический случай — *запросить и проверить*: спросить у пользователя значение имеет смысл хотя бы раз, а повторять — только если ввод плох.

```
   while / for            do-while
   ───────────            ────────
   проверить условие      выполнить тело
   ▼ (может 0 раз)        ▼ (минимум 1 раз)
   выполнить тело         проверить условие
                          ▼ повтор или выход
```

**Разобранные примеры.**

*Пример 1. Тело выполняется хотя бы раз.* Условие ложно сразу, но тело всё равно отработало один раз — сравните с примером 3 из §3.2.1:

```c
#include <stdio.h>

int main(void)
{
    int i = 10;
    do {
        printf("ran with i = %d\n", i);
        i++;
    } while (i < 5);
    return 0;
}
```

Вывод:

```
ran with i = 10
```

*Пример 2. Проверка ввода (валидация).* `do-while` идеально ложится на «повторяй запрос, пока значение неприемлемо». Здесь заодно проверяем и возврат `scanf` (§2.3.2), чтобы не зациклиться на нечисловом вводе:

```c
#include <stdio.h>

int main(void)
{
    int n = 0;
    int ok;
    do {
        printf("Enter a positive number: ");
        ok = scanf("%d", &n);
    } while (ok == 1 && n <= 0);
    printf("Got %d\n", n);
    return 0;
}
```

Если подать на вход `-3`, `0`, `8`, программа трижды запросит число и примет `8`:

```
Enter a positive number: Enter a positive number: Enter a positive number: Got 8
```

Приглашения без `\n` слились в строку из-за буферизации (§2.3.3) — на исполнение это не влияет: цикл отверг `-3` и `0` и завершился на `8`.

*Пример 3. Меню-цикл (`do-while` + `switch`).* Здесь сходятся почти все конструкции главы: цикл крутится, пока пользователь не выберет «выход», а `switch` разбирает команду:

```c
#include <stdio.h>

int main(void)
{
    int choice = -1;
    do {
        printf("Menu: 1=hello 2=bye 0=exit\n");
        if (scanf("%d", &choice) != 1)
            break;                        // некорректный ввод — выходим
        switch (choice) {
            case 1: printf("Hello!\n");  break;
            case 2: printf("Bye!\n");    break;
            case 0: printf("Exiting\n"); break;
            default: printf("Unknown\n");
        }
    } while (choice != 0);
    return 0;
}
```

При вводе `1`, `2`, `5`, `0`:

```
Menu: 1=hello 2=bye 0=exit
Hello!
Menu: 1=hello 2=bye 0=exit
Bye!
Menu: 1=hello 2=bye 0=exit
Unknown
Menu: 1=hello 2=bye 0=exit
Exiting
```

`5` попал в `default` («Unknown»), а `0` напечатал «Exiting» и сделал условие `choice != 0` ложным — цикл завершился.

**Связь с практикой.** Практическое правило выбора: **известно число повторений или есть счётчик — `for`; повторяем по условию без счётчика — `while`; тело обязано выполниться хотя бы раз (типично — «запросить и проверить») — `do-while`.** Меню и валидация ввода — «родная» территория `do-while`; чтение потока данных до конца — `while` (§3.2.5); перебор диапазона — `for`.

**Типичные ошибки и подводные камни.**

- **Забытая `;` после `while (условие)`** в `do-while` — ошибка компиляции. В отличие от `while`/`for`, здесь точка с запятой обязательна.
- **`do-while` там, где нужен `while`.** Если при негодном стартовом условии тело выполнять *нельзя* (например, обработать элемент, которого может не быть), `do-while` ошибочно выполнит его один раз. Тогда берите `while`.
- **Зацикливание на неверном вводе.** `do { scanf("%d", &n); } while (n <= 0);` без проверки возврата `scanf` зациклится на буквах: `scanf` не прочитает число, не тронет `n`, а «плохой» символ останется во входном буфере. Проверяйте возврат (как в примере 2).
- **Условие всегда истинно/ложно по ошибке.** Как и в `while`, следите, чтобы тело двигало условие к завершению.

**Источники:** C2 (гл. 6), C1 (гл. 3); нормативно о `do`-`while` и проверке условия после тела — C5.

### §3.2.4. `break`, `continue` и вложенные циклы

**Определения.**

- **`break`** внутри цикла немедленно завершает **самый внутренний** цикл (или `switch`), передавая управление на код *после* него. Оставшиеся итерации не выполняются.
- **`continue`** пропускает **остаток текущей итерации** и переходит к следующей: в `while`/`do-while` — сразу к проверке условия, в `for` — к части «изменение», затем к условию.
- **Вложенные циклы** — цикл в теле другого цикла. На каждый шаг внешнего цикла внутренний прокручивается полностью. `break`/`continue` действуют только на **тот** цикл, в котором записаны (обычно внутренний).

**Интуиция.** `break` — «аварийный выход» из цикла: условие ещё могло бы держаться, но мы уходим досрочно (нашли, что искали; случилась ошибка). `continue` — «пропустить этот случай и жить дальше»: текущий элемент не подходит, но цикл продолжается.

```
for (...) {
    ...
    if (плохо)  continue; ─┐  пропустить остаток итерации,
    ...                    │  перейти к следующей
    if (нашли)  break;   ──┼─▶ выйти из цикла целиком
    ...                    │
}   ◀──────────────────────┘
после цикла (сюда ведёт break)
```

**Разобранные примеры.**

*Пример 1. `break` — остановиться на первом найденном.* Ищем наименьший делитель числа; как только нашли — выходим (продолжать незачем):

```c
#include <stdio.h>

int main(void)
{
    int n = 91;
    int found = 0;
    for (int d = 2; d < n; d++) {
        if (n % d == 0) {
            found = d;
            break;                 // делитель найден — прекращаем поиск
        }
    }
    if (found)
        printf("%d is divisible by %d\n", n, found);
    else
        printf("%d is prime\n", n);
    return 0;
}
```

Вывод:

```
91 is divisible by 7
```

`91 = 7 × 13`; на `d = 7` условие сработало, `break` прервал цикл, не проверяя `d` до 90. `break` часто экономит работу: без него цикл прошёл бы до конца впустую.

*Пример 2. `continue` — пропустить неподходящие.* Суммируем только нечётные числа, «перепрыгивая» чётные:

```c
#include <stdio.h>

int main(void)
{
    int sum = 0;
    for (int i = 1; i <= 10; i++) {
        if (i % 2 == 0)
            continue;              // чётное — пропустить остаток итерации
        sum += i;
    }
    printf("sum of odds 1..10 = %d\n", sum);
    return 0;
}
```

Вывод:

```
sum of odds 1..10 = 25
```

Для чётных `i` `continue` пропускает `sum += i` и сразу выполняет `i++` (часть «изменение»). Суммируются 1+3+5+7+9 = 25. Тот же результат даёт `if (i % 2 != 0) sum += i;` — `continue` здесь вопрос стиля, но на длинном теле он спасает от глубокой вложенности.

*Пример 3. Вложенные циклы — таблица умножения.* Внешний цикл — строки, внутренний — столбцы:

```c
#include <stdio.h>

int main(void)
{
    for (int i = 1; i <= 3; i++) {
        for (int j = 1; j <= 3; j++)
            printf("%3d", i * j);       // ширина 3 — столбцы ровные
        printf("\n");                    // конец строки — после внутреннего цикла
    }
    return 0;
}
```

Вывод:

```
  1  2  3
  2  4  6
  3  6  9
```

На каждое значение `i` внутренний цикл полностью прокрутил `j` от 1 до 3. Спецификатор `%3d` (§2.3.1) выравнивает столбцы. Важно: `break` во внутреннем цикле прервал бы **только** его — внешний продолжил бы работу.

**Связь с практикой.** `break` — стандартный выход из поиска и способ разорвать «вечный» цикл `while (1)` по условию внутри. `continue` убирает вложенность, отсекая неподходящие случаи в начале тела. Вложенные циклы — основа обработки таблиц и, позже, двумерных массивов (глава 5) и сравнения «каждого с каждым». Помните ограничение: и `break`, и `continue` управляют лишь ближайшим циклом; выйти сразу из нескольких вложенных штатных средств у C нет (обычно помогает флаг — §3.2.5).

**Типичные ошибки и подводные камни.**

- **Ожидать, что `break` выйдет из всех вложенных циклов.** Он выходит только из внутреннего. Чтобы прервать внешний, используют флаг (`done = 1; break;` и проверку `if (done) break;` во внешнем) — приём из §3.2.5.
- **`continue` в `while` пропускает шаг.** В `while` `continue` идёт сразу к условию; если шаг (`i++`) стоял *после* места `continue`, он будет пропущен — рискуете бесконечным циклом. В `for` часть «изменение» выполняется всегда, поэтому такой ловушки нет.
- **`break` в `switch` внутри цикла.** `break` завершит `switch`, а **не** цикл. Чтобы выйти из цикла из-под `switch`, нужен флаг и `break` уже в цикле.
- **`continue`, маскирующий отсутствие шага.** Убедитесь, что при `continue` цикл всё равно движется к завершению.

**Источники:** C2 (гл. 6), C1 (гл. 3); нормативно о `break`, `continue` и их области действия — C5.

### §3.2.5. Классические паттерны циклов

**Определения.** За разнообразием циклов стоит небольшой набор **паттернов** — типовых схем, из которых собирается большинство обработки данных. Освоив их, вы будете «видеть» решение задачи как комбинацию знакомых шаблонов.

- **Счётчик** — переменная, считающая события (`count++` при выполнении условия).
- **Накопитель** — переменная, вбирающая результат по частям: сумма (старт 0), произведение (старт 1), максимум/минимум (старт — первое значение).
- **Флаг** — целая переменная-«признак» (0/1), фиксирующая факт («нашли», «всё отсортировано») для решения после цикла.
- **Поиск** — цикл с `break` по факту нахождения (§3.2.4).
- **Чтение потока** — обработка заранее неизвестного количества значений: `while (scanf(...) == 1) { … }` крутится, пока `scanf` успешно читает, и завершается на конце ввода или на нечисловом символе (§2.3.2).

**Интуиция.** Все паттерны — вариации одной идеи: **до цикла завести переменную-итог в нейтральном состоянии, в теле обновлять её по одному элементу, после цикла воспользоваться результатом.**

```
   ДО:    завести итог  (sum=0 / prod=1 / max=?  / found=0 / count=0)
   ┌──────────────────────────────────────────────┐
   │  на каждый элемент:  обновить итог             │  ← цикл
   └──────────────────────────────────────────────┘
   ПОСЛЕ: использовать итог (напечатать, сравнить, решить)
```

**Разобранные примеры.**

*Пример 1. Статистика потока: счётчик + накопитель + максимум + среднее.* Читаем неизвестное число целых до конца ввода и одним проходом собираем сразу несколько итогов:

```c
#include <stdio.h>

int main(void)
{
    int x;
    int count = 0;
    long long sum = 0;
    int max = 0;                       // будет переопределён первым числом
    while (scanf("%d", &x) == 1) {
        if (count == 0 || x > max)     // первое число задаёт максимум
            max = x;
        sum += x;
        count++;
    }
    if (count == 0) {
        printf("no numbers\n");
    } else {
        printf("count = %d\n", count);
        printf("sum   = %lld\n", sum);
        printf("max   = %d\n", max);
        printf("avg   = %.2f\n", (double)sum / count);   // вещественное деление, §2.2.5
    }
    return 0;
}
```

При вводе `4 8 15 16 23 42`:

```
count = 6
sum   = 108
max   = 42
avg   = 18.00
```

Здесь сразу три паттерна: **счётчик** `count`, **накопитель** `sum`, **бегущий максимум** `max`. Приём `count == 0 || x > max` корректно инициализирует максимум первым же числом (иначе стартовое `max = 0` было бы неверным для потока из одних отрицательных). Приведение `(double)sum` (§2.2.5) обязательно — иначе среднее посчиталось бы целочисленно. И `count == 0` в конце страхует от деления на ноль при пустом вводе.

*Пример 2. Флаг: проверка на простоту.* Флаг `is_prime` хранит «пока не опровергнутую» гипотезу; цикл ищет опровержение (делитель) и при находке опускает флаг и выходит:

```c
#include <stdio.h>

int main(void)
{
    int n = 29;
    int is_prime = 1;                  // гипотеза: число простое
    if (n < 2) {
        is_prime = 0;
    } else {
        for (int d = 2; d < n; d++) {
            if (n % d == 0) {
                is_prime = 0;          // нашли делитель — гипотеза опровергнута
                break;
            }
        }
    }
    if (is_prime)
        printf("%d is prime\n", n);
    else
        printf("%d is not prime\n", n);
    return 0;
}
```

Вывод:

```
29 is prime
```

Флаг — стандартный способ «донести» результат цикла до кода после него, а также аккуратно выйти из вложенных циклов (опустить флаг + `break`, §3.2.4).

*Пример 3. Разбор числа по разрядам (`%` и `/` в цикле).* Пара «остаток от деления на 10 / деление на 10» из §2.2.1 позволяет перебрать цифры числа справа налево — здесь считаем их сумму:

```c
#include <stdio.h>

int main(void)
{
    int n = 1234;
    int original = n;
    int digit_sum = 0;
    while (n > 0) {
        digit_sum += n % 10;           // последняя цифра
        n /= 10;                        // отбросить её
    }
    printf("digit sum of %d = %d\n", original, digit_sum);
    return 0;
}
```

Вывод:

```
digit sum of 1234 = 10
```

`n` проходит 1234 → 123 → 12 → 1 → 0, отдавая цифры 4, 3, 2, 1; их сумма 10. Тот же приём (`% 10`, `/= 10`) переворачивает число, считает количество цифр, переводит в другую систему счисления.

**Связь с практикой.** Эти пять паттернов покрывают львиную долю «обработки данных» до появления массивов: подсчёт статистики ввода, валидация, поиск, разбор чисел. Особенно важен **`while (scanf(...) == 1)`** — это каноничный способ прочитать поток данных неизвестной длины (файл чисел, ввод пользователя до «конца»), к которому мы вернёмся при работе с файлами (глава 10). Научившись комбинировать счётчик, накопитель и флаг, вы решаете задачи, которые ещё главу назад казались требующими «запомнить все числа», — без всякого хранилища, одним проходом.

**Типичные ошибки и подводные камни.**

- **Неверная инициализация накопителя.** Сумма должна стартовать с 0, произведение — с 1, максимум/минимум — с *первого* элемента (а не с 0: для потока отрицательных `max = 0` даст неверный ответ). Приём — флаг «первый элемент» или `count == 0`.
- **Целочисленное среднее.** `sum / count` при целых — целочисленное деление (§2.2.5); для дробного среднего приводите: `(double)sum / count`.
- **Забытая проверка пустого ввода.** Деление на `count` при `count == 0` — деление на ноль (§2.2.1). Проверяйте перед делением.
- **Игнорирование возврата `scanf`.** `while (scanf("%d", &x))` вместо `== 1` может неверно трактовать конец ввода и нечисловые символы. Сравнивайте с числом успешно прочитанных элементов (`== 1`).
- **Переполнение накопителя.** Сумма/произведение многих или больших чисел легко выходит за `int` (§2.1.2); для сумм берите `long long`, как в примере 1.

**Источники части — см. ниже.** Нормативно о значении `scanf` — C5, C6; паттерны и дидактика — C2 (гл. 6), C7 (нед. 1).

## Резюме главы

- **Ветвление** выбирает, *какой* код выполнить. `if`/`else` разветвляет по условию; условие ложно при значении `0` и истинно при любом ненулевом (истинность = «ноль/не ноль», §3.1.1). Цепочка `else if` выбирает одну ветвь из многих — порядок условий важен (частные выше общих). `else` связывается с ближайшим `if` — отсюда «висячий else», от которого спасают фигурные скобки.
- **`switch`** — множественный выбор по значению целочисленного выражения: вход по метке `case`, выход по `break`; без `break` управление «проваливается» в следующую ветвь (иногда полезно для группировки, чаще — ошибка, которую ловит `-Wimplicit-fallthrough`). Для диапазонов `switch` не подходит — там `else if`.
- **Тернарный оператор** `?:` — это `if`/`else` как *выражение* со значением; хорош для коротких «одно из двух значений», плох для логики и глубокой вложенности.
- **Циклы** повторяют код. `while` и `for` проверяют условие *до* тела (могут не выполнить его); `do-while` — *после* (выполнит минимум раз). `for` собирает инициализацию/условие/шаг в заголовок (удобен при известном числе повторений), `while` оставляет управление в теле (удобен «по условию»), `do-while` — для «запросить и проверить». У любого цикла три части: инициализация, условие продолжения, шаг к завершению; нет шага — бесконечный цикл.
- **`break`** досрочно выходит из ближайшего цикла, **`continue`** пропускает остаток итерации; оба действуют только на свой (внутренний) цикл. Вложенные циклы прокручивают внутренний полностью на каждый шаг внешнего.
- **Паттерны:** счётчик, накопитель (сумма — старт 0, произведение — 1, макс/мин — первое значение), флаг, поиск с `break`, чтение потока `while (scanf(...) == 1)`. Общая схема: завести итог до цикла, обновлять в теле, использовать после. Классические ловушки: off-by-one, бесконечный цикл, целочисленное среднее, деление на ноль при пустом вводе, переполнение накопителя.

**Дальше.** Теперь программа умеет выбирать путь и повторять действия — но весь код по-прежнему свален в одну функцию `main`, и повторно использовать кусок логики (например, «проверить, простое ли число») можно лишь копированием. Глава 4 вводит **функции**: способ дать участку кода имя, параметры и результат, вызывать его откуда угодно и рассуждать о программе как о наборе «контрактов». Заодно мы впервые заглянем в **стек вызовов** — модель, на которую обопрутся указатели и динамическая память (главы 6–7).
