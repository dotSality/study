## 3. Правила использования реестра (из плана, разделы 4.3–4.4)

1. **Каждая часть каждой главы сверяется минимум с двумя независимыми источниками** из этого реестра; идентификаторы источников (C1…C12, R1…R11) проставляются в конце каждой части в блоке «Источники». Исключение по гл. 26 (единственный источник по egui) оговорено в строке R11 и в `outline.md`.
2. Ресурсы с пометкой 🔶 не могут быть единственной опорой части — только в паре с академическим/официальным источником.
3. При расхождении формулировок арбитром выступает нормативный источник: **C5 (стандарт C)** для тома I, **R7 (Rust Reference)** для тома II; альтернативная терминология оговаривается явно.
4. Прямое копирование текста запрещено — только пересказ/синтез со ссылкой.
5. **Запрет на догадки (п. 4.4 плана, абсолютное правило):** утверждение, которое не удалось подтвердить по источникам реестра (при необходимости — через `/sc:research`), в текст не попадает; пропуск фиксируется как открытый вопрос в `outline.md`.
6. Найденный в ходе написания новый источник добавляется в реестр **до** использования, с проверкой по критериям п. 4.2 плана и отметкой даты проверки.

## 4. Открытые вопросы реестра

- Русскоязычные издания перечисленных книг существуют (например, переводы K&R и «The Book»), но их актуальность изданиям-оригиналам не проверялась; до проверки в реестр не включены — терминологию на русском выверять по оригиналу с приведением английского термина в скобках (п. 7 плана).
- Для главы 24 (итоговые проекты) допускается точечное расширение реестра источниками по конкретной предметной области проекта (например, сетевое программирование) — по правилу 6 выше.
